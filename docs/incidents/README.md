# Incidents

## Date

2026-06-14 Sunday

## Purpose

This directory stores incident reports and postmortems.

An incident is any unplanned event that negatively impacts service availability, reliability, security, or operations.

Examples:

* Minecraft outage
* Backup failure
* Disk health warning
* Network outage
* Failed deployment
* Service misconfiguration

---

## Goals

Incident documentation helps:

* Improve troubleshooting skills
* Identify recurring problems
* Build operational maturity
* Create portfolio-worthy learning material

---

## Incident Report Naming Convention

Example:

incident-2026-07-minecraft-outage.md

incident-2026-09-backup-failure.md

---

## Postmortem Philosophy

The goal is learning rather than blame.

Each incident should answer:

* What happened?
* Why did it happen?
* How was it resolved?
* How can it be prevented in the future?
